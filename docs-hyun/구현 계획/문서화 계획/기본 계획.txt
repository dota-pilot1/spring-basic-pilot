[백엔드 API 및 시스템 문서화 계획]

프로젝트가 고도화됨에 따라 협업과 유지보수를 위한 문서화는 필수적입니다. 현재 단계에서 진행할 수 있는 효과적인 문서화 방안입니다.

1. API 자동 문서화 (SpringDoc OpenAPI / Swagger)
- 목적: 코드를 수정하면 문서가 자동으로 업데이트되도록 설정. 프론트엔드 개발자가 API 명세서를 즉시 확인할 수 있음.
- 실천: `springdoc-openapi` 라이브러리를 `build.gradle`에 추가.
- 결과물: `http://localhost:9090/swagger-ui.html` 접속 시 웹에서 API 테스트 및 명세 확인 가능.

2. API 테스트 명세서 (Postman Collection)
- 목적: Postman에서 만든 테스트 케이스(가입, 검색, 게시물 생성 등)를 팀원과 공유.
- 실천: Postman에서 `Export`를 통해 JSON 파일로 저장 후 프로젝트의 `docs-hyun/api-test/` 폴더에 관리.

3. 데이터베이스 설계서 (ERD)
- 목적: 테이블 간의 관계(유저-게시글)를 한눈에 파악.
- 실천: DBeaver의 ER Diagram 기능을 활용하거나 Mermaid 문법으로 `docs-hyun/db/erd.md` 작성.

4. 비즈니스 로직 시퀀스 다이어그램
- 목적: 가입 로직이나 페이징 로직 등 복잡한 흐름을 시각화.
- 실천: Mermaid 문법(Markdown)을 사용하여 핵심 로직의 흐름도 작성.
